      Enable "Try It Out" for your API

      • Your API specifications must be based on Open API Specification (OAS) 2.0.
      • You must have Write permissions to create an API.
      1. Add a CORS Restriction policy.
        You can add the CORS Restriction policy for:
        • All APIs in your Developer Portal. Learn how to enable Try It Out in the Developer Portal settings.
        • A specific API. Learn how to add and configure the CORS Restriction policy for a specific API.
        Note: If the CORS Restriction policy is set up for the entire Developer Portal and for the specific API, the policy for the specific API has precedence over the policy for the entire Developer Portal.
      2. Publish your API.
      3. Test the Try it out button for your API.
