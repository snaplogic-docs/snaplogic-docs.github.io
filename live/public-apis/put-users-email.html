PUT /users/{email}

updates the user record associated with the specified email address.

Overview

This API updates the user record associated with the specified email address.

Prerequisites

  • Org Admin permissions
  • Request
  • Response
  • Notes
  • Examples
   PUT https://{pod_path}/api/1/rest/public/users/{email}

Path Parameters

Key Description
pod_path Required. The path to your SnapLogic pod.

Example: elastic.snaplogic.com
email Required. The email address of the user.

Query Parameters

None.

Request Header

Specify Basic Auth for authorization.

   Authorization: Basic {your_encoded_security_credentials}

Request Body

   {
    "first_name" : "...",
    "last_name" : "...",
    "allow_password_login" : ...,
    "revoke_old_password" : ...,
    "utility" : ...,
    "ui_access" : ...,
    "created_by" : "...",
    "certificate" : "...",
    "file_handle" : "..."
  }

where:

Key Type Description
first_name string The first name of the user.
last_name string The last name of the user.
allow_password_login Boolean If true, ???
revoke_old_password Boolean If true, ???
utility Boolean If true, ???
ui_access Boolean If true, the user is granted access to the SnapLogic UI.
created_by string ???
certificate string ???
file_handle string ???

Response Header

Key Description
X-SL-StatusCode Status code indicating the results of the request.

Learn more: HTTP Status Code Registry
